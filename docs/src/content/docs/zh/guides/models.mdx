## 默认模型

当初始化 `Agent` 时未指定模型，将使用默认模型。当前默认是出于兼容性与低延迟考虑的 [`gpt-4.1`](https://platform.openai.com/docs/models/gpt-4.1)。如果你有权限，建议将智能体设置为 [`gpt-5.2`](https://platform.openai.com/docs/models/gpt-5.2) 以获得更高质量，同时显式保留 `modelSettings`。

如果你想切换到其他模型（如 [`gpt-5.2`](https://platform.openai.com/docs/models/gpt-5.2)），有两种方式配置你的智能体。

首先，如果你希望对所有未设置自定义模型的智能体一致地使用某个特定模型，请在运行智能体之前设置环境变量 `OPENAI_DEFAULT_MODEL`。

```bash
export OPENAI_DEFAULT_MODEL=gpt-5.2
node my-awesome-agent.js
```

其次，你可以为某个 `Runner` 实例设置默认模型。如果未为智能体设置模型，将使用该 `Runner` 的默认模型。

### GPT-5.x 模型

当你以这种方式使用任意 GPT-5.x 模型（如 [`gpt-5.2`](https://platform.openai.com/docs/models/gpt-5.2)）时，SDK 会应用默认的 `modelSettings`，其设置对大多数用例效果最佳。若要调整默认模型的推理强度，请传入你自己的 `modelSettings`：

若需更低延迟，推荐在 `gpt-5.2` 中使用 `reasoning.effort: "none"`。gpt-4.1 系列（包括 mini 和 nano 变体）同样是构建交互式智能体应用的稳健选择。

如果传入非 GPT-5 的模型名称且未提供自定义 `modelSettings`，SDK 会回退到适用于任意模型的通用 `modelSettings`。

## OpenAI 提供方

默认的 `ModelProvider` 使用 OpenAI API 解析名称。它支持两个不同的端点：

| API              | 用途                                             | 调用 `setOpenAIAPI()`               |
| ---------------- | ------------------------------------------------ | ----------------------------------- |
| Chat Completions | 标准对话与函数调用                               | `setOpenAIAPI('chat_completions')`  |
| Responses        | 全新以流式为先的生成式 API（工具调用、灵活输出） | `setOpenAIAPI('responses')`（默认） |

### Responses WebSocket 传输

当你将 OpenAI 提供方与 Responses API 搭配使用时，可以通过 WebSocket 传输发送请求，而不是默认的 HTTP 传输。

可使用 `setOpenAIResponsesTransport('websocket')` 全局启用，或通过 `new OpenAIProvider({ useResponses: true, useResponsesWebSocket: true })` 为单个提供方启用。

仅使用 WebSocket 传输并不需要 `withResponsesWebSocketSession(...)` 或自定义 `OpenAIProvider`。如果每次运行/请求都重新连接可以接受，在启用 `setOpenAIResponsesTransport('websocket')` 后，你现有的 `run()` / `Runner.run()` 用法将继续正常工作。

仅当你希望优化连接复用并更明确地管理 websocket 提供方生命周期时，才使用 `withResponsesWebSocketSession(...)` 或自定义 `OpenAIProvider` / `Runner`：

- `withResponsesWebSocketSession(...)`：便捷的作用域生命周期，在回调后自动清理。
- 自定义 `OpenAIProvider` / `Runner`：在你的应用架构中进行显式的生命周期控制（包括关闭清理）。

尽管名称如此，`withResponsesWebSocketSession(...)` 是一个传输层生命周期辅助工具，与[会话](/openai-agents-js/zh/guides/sessions)指南中描述的内存 `Session` 接口无关。

如果你使用 websocket 代理或网关，请在 `OpenAIProvider` 上配置 `websocketBaseURL`，或设置 `OPENAI_WEBSOCKET_BASE_URL`。

参见 [`examples/basic/stream-ws.ts`](https://github.com/openai/openai-agents-js/tree/main/examples/basic/stream-ws.ts) 以获取使用 Responses WebSocket 传输的完整流式传输 + 人工干预（HITL）示例。

## ModelSettings

`ModelSettings` 与 OpenAI 的参数一一对应，但与提供方无关。

| 字段                   | 类型                                                            | 说明                                                                      |
| ---------------------- | --------------------------------------------------------------- | ------------------------------------------------------------------------- |
| `temperature`          | `number`                                                        | 创造性与确定性的权衡。                                                    |
| `topP`                 | `number`                                                        | 核采样。                                                                  |
| `frequencyPenalty`     | `number`                                                        | 惩罚重复的 token。                                                        |
| `presencePenalty`      | `number`                                                        | 鼓励生成新 token。                                                        |
| `toolChoice`           | `'auto' \| 'required' \| 'none' \| string`                      | 参见[强制工具使用](/openai-agents-js/zh/guides/agents#forcing-tool-use)。 |
| `parallelToolCalls`    | `boolean`                                                       | 在支持的情况下允许并行函数调用。                                          |
| `truncation`           | `'auto' \| 'disabled'`                                          | token 截断策略。                                                          |
| `maxTokens`            | `number`                                                        | 响应中的最大 token 数。                                                   |
| `store`                | `boolean`                                                       | 持久化响应，便于检索/RAG 工作流。                                         |
| `promptCacheRetention` | `'in-memory' \| '24h' \| null`                                  | 在受支持时控制提供方提示词缓存的保留策略。                                |
| `reasoning.effort`     | `'none' \| 'minimal' \| 'low' \| 'medium' \| 'high' \| 'xhigh'` | 适用于 gpt-5.x 模型的推理强度。                                           |
| `reasoning.summary`    | `'auto' \| 'concise' \| 'detailed'`                             | 控制模型返回的推理摘要量。                                                |
| `text.verbosity`       | `'low' \| 'medium' \| 'high'`                                   | 适用于 gpt-5.x 等的文本冗长度。                                           |
| `providerData`         | `Record<string, any>`                                           | 传递给底层模型的提供方特定直通选项。                                      |

可在任一层级附加这些设置：

`Runner` 级别的设置会覆盖任何冲突的按智能体设置。

## 提示词（Prompt）

可以通过 `prompt` 参数为智能体配置一个服务端存储的提示词配置，用于控制智能体行为。目前，仅当你使用 OpenAI 的
[Responses API](https://platform.openai.com/docs/api-reference/responses) 时支持此选项。

| 字段        | 类型     | 说明                                                                              |
| ----------- | -------- | --------------------------------------------------------------------------------- |
| `promptId`  | `string` | 提示词的唯一标识符。                                                              |
| `version`   | `string` | 你希望使用的提示词版本。                                                          |
| `variables` | `object` | 代入提示词中的键/值变量。值可以是字符串，或内容输入类型（例如文本、图片或文件）。 |

任何额外的智能体配置（如 tools 或 instructions）都会覆盖你在已存储提示词中可能配置的值。
