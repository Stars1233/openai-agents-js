## サポートされる環境

OpenAI Agents SDK は次のサーバー環境でサポートされています:

- Node.js 22+
- Deno 2.35+
- Bun 1.2.5+

### 限定サポート

- **Cloudflare Workers**: Agents SDK は Cloudflare Workers で使用できますが、現時点ではいくつかの制限があります:
  - 現在の SDK は `nodejs_compat` を有効にする必要があります
  - トレースはリクエストの最後に手動でフラッシュする必要があります。詳細は [トレーシング](/openai-agents-js/ja/guides/tracing#export-loop-lifecycle) を参照してください
  - Cloudflare Workers の `AsyncLocalStorage` サポートが限定的なため、一部のトレースが正確でない可能性があります
  - アウトバウンド WebSocket 接続は fetch ベースのアップグレードを使用する必要があります（グローバルの `WebSocket` コンストラクタは不可）。Realtime を使用する場合は、`@openai/agents-extensions` の Cloudflare トランスポート（`CloudflareRealtimeTransportLayer`）を使用してください
- **Responses API WebSocket transport**:
  - グローバルな `WebSocket` 実装が必要です
  - ハンドシェイク時のカスタムヘッダーをサポートする `WebSocket` 実装が必要です
  - 多くのブラウザスタイルの WebSocket API（および一部のエッジランタイム）は、アウトバウンドのカスタムヘッダーをサポートしていません。そのような環境では、代わりにデフォルトの HTTP Responses トランスポートを使用してください
  - グローバルな `WebSocket` 実装が見つからない、またはカスタムヘッダーのサポートがないことを示すエラーが表示される場合、そのランタイムの WebSocket 実装は Responses WebSocket トランスポートと互換性がありません
- **Browsers**:
  - ブラウザでは現在トレーシングはサポートされていません
- **v8 isolates**:
  - 適切なブラウザポリフィルを備えたバンドラを使用すれば v8 isolates 向けに SDK をバンドルできますが、トレーシングは動作しません
  - v8 isolates での動作は広範にはテストされていません

## デバッグロギング

SDK で問題が発生している場合は、デバッグロギングを有効にして詳細な情報を取得できます。

`DEBUG` 環境変数を `openai-agents:*` に設定してデバッグロギングを有効にします。

```bash
DEBUG=openai-agents:*
```

または、SDK の特定の部分にデバッグをスコープできます:

- `openai-agents:core` — SDK の主要な実行ロジック
- `openai-agents:openai` — OpenAI API 呼び出し
- `openai-agents:realtime` — Realtime Agents のコンポーネント
